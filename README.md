# 📁 Re_View — 프로젝트 문서 레포지토리

본 레포지토리는 **Re_View(리뷰 기반 스킨케어 추천 서비스)** 프로젝트의 기획 및 설계 문서를 모아놓은 저장소입니다.

- 코드 레포지토리: https://github.com/VenyVince/Re_View
- 시연 영상: https://www.youtube.com/watch?v=kP2HrcrGmvU

---

## 📄 문서 목록

| 문서명 | 설명 |
|--------|------|
| 기획안 | 서비스 목표, 주요 기능 정의, 팀 역할 분담 등 프로젝트 전반의 방향성을 담은 기획 문서 |
| WBS | 전체 작업 항목, 담당자, 기간, 완료율을 정리한 프로젝트 일정 관리 문서 |
| 수행일지 | 주차별 진행 상황, 이슈 및 해결 내용을 기록한 프로젝트 진행 일지 |
| 포트폴리오(PPT) | 프로젝트 개요, 기술 선택 근거, 주요 기능 및 결과를 정리한 최종 발표 자료 |
| 멘토링 피드백 | 멘토링 세션에서 받은 피드백과 이를 반영한 개선 사항 기록 |

## 👥 팀 구성 및 역할

| 이름 | 역할 | 담당 영역 |
|------|------|-----------|
| 김석현 | 팀장 / 백엔드·인프라 | DB 설계, 회원·포인트·결제수단·장바구니·이미지 처리·MinIO·배포 |
| 이재빈 | 백엔드 | 로그인·결제·구매·추천 알고리즘 |
| 정윤성 | 백엔드 | 상품·리뷰·QnA·커뮤니티 |
| 오승환 | 프론트엔드 | 마이페이지 전체 UI |
| 김시연 | 프론트엔드 | 관리자 페이지·설문조사 UI |
| 박진성 | 프론트엔드 | 검색·상품·리뷰 페이지 UI |
